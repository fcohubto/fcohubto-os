Architecture Spec

Las tres capas del sistema y el flujo oficial de información entre ellas. v1 · Mayo 2026

01 Francisco System Capa personal
La capa raíz. Donde viven la identidad, las decisiones estratégicas y el criterio que no se delega.
  • Definición de norte y prioridades personales
  • Gobernanza de todo el sistema (qué existe, qué se archiva, qué escala)
  • Repositorio: fcohubto-os (este dashboard)
↓
02 OLIVA OS Capa de pensamiento
El sistema de IA estructurado. Convierte el criterio de Francisco en prompts, frameworks y procesos reproducibles.
  • Modelos R (Razonamiento), C (Creación), P (Producción)
  • Biblioteca de prompts versionados
  • Repositorio: fw_oliva
↓
03 Olivacraft Capa de negocio
El estudio. Donde los sistemas se convierten en servicio real a clientes reales.
  • Operación de proyectos (flujo, contrato, entrega)
  • Comunicación y propuestas al mercado
  • Repositorio: olivacraftos

El criterio fluye de arriba hacia abajo. El feedback del negocio sube como aprendizaje, no como modificación directa de la capa superior.

Reglas de gobernanza — qué NO puede pasar
✕ Información de clientes de Olivacraft no puede subir a OLIVA OS ni a Francisco System sin anonimizar.
✕ Lógica de negocio (precios, contratos, decisiones comerciales) no puede residir en OLIVA OS — pertenece a Olivacraft.
✕ Prompts de OLIVA OS no pueden referenciar nombres de clientes reales ni datos operativos de Olivacraft.
✕ Decisiones de identidad (quién es Francisco, qué hace el sistema) no se toman desde Olivacraft hacia arriba — solo desde Francisco System hacia abajo.

OLIVA OS

El framework de pensamiento y ejecución con IA que conecta a Francisco con Olivacraft.

Framework OLIVA

El sistema de prompts, modelos y gobernanza que convierte el criterio de Francisco en procesos reproducibles. Tres modelos: R para pensar, C para crear, P para producir.

Abrir OLIVA OS →
Estable · pendiente cargar prompts nivel 9
R
Razonamiento
Análisis, diagnóstico, estructuración de problemas complejos. Se activa antes de ejecutar cualquier tarea no trivial.
C
Creación
Generación de contenido, diseño conceptual, comunicación. Opera desde criterio previo, no desde cero.
P
Producción
Ejecución directa y entregables finales. Velocidad con calidad garantizada por el sistema previo.
Estado actual
Estable
Repositorio activo en fw_oliva. Estructura base de los tres modelos publicada. Prompts de nivel 9 en proceso de carga — pendiente de versionado formal.
Relación con el sistema
OLIVA OS recibe criterio desde Francisco System y entrega frameworks y prompts a Olivacraft. Es la capa de transformación — convierte visión en proceso ejecutable.
